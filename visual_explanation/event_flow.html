DAOOB New Event Request Flow

The new event request feature creates a more streamlined experience where clients submit requests to admins who coordinate with vendors.

Mobile App Flow

Step 1: Home Screen
Client taps the prominent "Plan a New Event" button on home screen
→
Step 2: Event Type Selection
Client browses and selects an event type (Wedding, Birthday, etc.) from a grid of options
→
Step 3: Questionnaire
Client answers event-specific questions about guest count, venue preferences, budget, etc.
Step 4: Review & Submit
Client reviews their answers and submits the request
→
Step 5: Confirmation
Client sees confirmation that request was sent to admin
→
Step 6: Notification
Client receives notification when admin responds with quotation

Admin Web Dashboard Flow

Step 1: Event Requests Tab
Admin sees new event requests in the Events section of dashboard
→
Step 2: Review Request
Admin reviews client requirements and event details
→
Step 3: Create Quotation
Admin prepares pricing and options

Note: The original vendor browsing and direct booking functionality is still available if clients prefer to work directly with vendors.
